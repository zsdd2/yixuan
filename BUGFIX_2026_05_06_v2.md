# Bug 修复记录 - 2026-05-06

## 问题1：导入中心图片加载404 ✅ 已解决

### 问题描述
- 导入中心的照片缩略图无法加载，显示404错误
- 影响范围：207张照片（53.6%）

### 根本原因
历史数据污染：207张照片的 `original_path` 和 `thumbnail_path` 路径错误
- `original_path` 指向不存在的缩略图文件
- `thumbnail_path` 指向存在的缩略图文件
- 前端优先使用 `thumbnail_path`，导致加载失败

### 修复方案
执行 SQL 互换路径：
```sql
UPDATE photos 
SET 
  original_path = thumbnail_path,
  thumbnail_path = original_path
WHERE original_path LIKE '%/thumb/%';
```

### 修复结果
- ✅ 207张照片路径已修复
- ✅ 所有386张照片的缩略图现在都可以正常访问（HTTP 200）
- ✅ 前端回退逻辑正常工作（`thumbnail_path || original_path`）

## 问题2：移除出项目功能失效 ⚠️ 需要用户配合诊断

### 问题描述
用户反馈"导入中心，移除出项目失效，无法移除"

### 代码验证
- ✅ 前端代码正确：`request.patch('/api/v1/photos/bulk-update', { photo_ids: ids, remove_from_target: true })`
- ✅ 后端代码正确：`photo.target_id = None`（photos.py:561-562）
- ✅ 数据库操作正常：手动测试成功

### 可能原因
1. **Token 过期**：Docker 日志显示 `401 Unauthorized` 错误
2. **浏览器缓存**：旧的请求可能被缓存

### 诊断步骤
请用户执行以下操作：

1. **强制刷新页面**
   - 按 `Ctrl+Shift+R`（Windows）或 `Cmd+Shift+R`（Mac）
   - 如果自动跳转到登录页，重新登录

2. **检查 Token**
   - 打开浏览器开发者工具（F12）
   - 切换到 Console 标签
   - 执行：`localStorage.getItem('token')`
   - 如果返回 `null`，说明需要重新登录

3. **查看网络请求**
   - 打开开发者工具（F12）
   - 切换到 Network 标签
   - 选择照片，点击「移除出项目」
   - 查看 `bulk-update` 请求：
     - 状态码应该是 200（成功）
     - 如果是 401，说明 Token 过期
     - 如果是其他错误，查看 Response 内容

4. **验证功能**
   - 重新登录后，选择照片
   - 点击「移除出项目」
   - 检查照片是否从列表中消失
   - 刷新页面验证照片确实被移除

### 临时解决方案
如果问题持续，可以手动执行 SQL：
```sql
-- 将照片143从目标4中移除
UPDATE photos SET target_id = NULL WHERE id = 143;
```
