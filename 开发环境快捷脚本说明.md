# 开发环境快捷脚本说明

## 问题背景

在开发过程中，经常遇到以下问题：
- 修改代码后数据显示异常
- 页面一直显示加载中（骨架屏）
- API 请求失败或返回旧数据
- 前端端口漂移（15173 → 15174 → 15175）
- 数据库连接异常

**根本原因：**
1. 后端热重载后数据库连接池未正确重置
2. 前端 HMR 热更新不完全，缓存了旧数据
3. 浏览器缓存了旧的 API 响应
4. 端口被占用导致前端服务启动在错误端口

## 快捷脚本

### 1. `restart_dev.bat` - 快速重启（保留数据）

**适用场景：**
- 修改前端组件后，页面未更新
- 修改后端代码后，接口行为异常
- 前端端口漂移到 15174/15175
- 数据显示异常但不想清空数据

**执行内容：**
1. 清理前端端口（15173/15174/15175）
2. 重启后端容器
3. 启动前端服务

**使用方法：**
```bash
# 双击运行或在命令行执行
restart_dev.bat
```

---

### 2. `restart_with_db.bat` - 完全重启（重置数据库）

**适用场景：**
- 修改数据库模型（`app/models.py`）后
- 数据库连接异常或数据损坏
- 接口返回 500 错误
- 需要清空所有数据重新开始

**执行内容：**
1. 清理前端端口
2. 停止后端容器
3. **重置数据库（清空所有数据）**
4. 启动后端容器
5. 启动前端服务

**使用方法：**
```bash
# 双击运行或在命令行执行
restart_with_db.bat
```

**⚠️ 警告：此脚本会清空所有数据！**

## 使用建议

### 日常开发流程

1. **修改前端代码后：**
   ```bash
   restart_dev.bat
   ```

2. **修改后端代码后：**
   - 如果只是修改业务逻辑：等待自动重启（uvicorn --reload）
   - 如果修改数据库模型：运行 `restart_with_db.bat`

3. **遇到数据异常时：**
   - 先尝试 `restart_dev.bat`
   - 如果问题依然存在，运行 `restart_with_db.bat`

4. **每天开始工作前：**
   ```bash
   restart_dev.bat  # 确保环境干净
   ```

### 故障排查流程

```
遇到问题
    ↓
强制刷新浏览器（Ctrl+Shift+R）
    ↓
问题解决？ → 是 → 继续开发
    ↓ 否

    ↓
问题解决？ → 是 → 继续开发
    ↓ 否
运行 restart_with_db.bat
    ↓
问题解决？ → 是 → 继续开发
    ↓ 否
```

## 手动重启命令

如果快捷脚本无法使用，可以手动执行以下命令：

### 仅重启前端
```bash
npx kill-port 15173 15174 15175
cd frontend_admin
npm run dev
```

### 仅重启后端
```bash
docker-compose restart web
```

### 完全重启（含数据库重置）
```bash
# 1. 清理端口
npx kill-port 15173 15174 15175

# 2. 停止后端
docker-compose stop web

# 3. 重置数据库
python reset_db.py

# 4. 启动后端
docker-compose start web

# 5. 启动前端
cd frontend_admin
npm run dev
```

## 常见问题

### Q1: 脚本运行后前端无法访问？
**A:** 检查前端是否成功启动：
```bash
netstat -ano | findstr ":15173"
```
如果没有输出，手动进入 `frontend_admin` 目录运行 `npm run dev`

### Q2: 后端容器无法启动？
**A:** 查看容器日志：
```bash
docker-compose logs -f web --tail=50
```
检查是否有 Python 语法错误或依赖缺失

### Q3: 数据库重置后无法启动？
**A:** 检查数据库容器状态：
```bash
docker-compose ps
docker-compose logs -f db --tail=50
```

### Q4: 脚本执行到一半卡住？
**A:** 按 `Ctrl+C` 中断，然后手动执行各步骤命令

## 技术细节

### 为什么需要清理端口？
Vite 开发服务器在端口被占用时会自动跳转到下一个端口（15173 → 15174 → 15175），导致前端代理配置失效，API 请求无法正确转发到后端。

### 为什么需要重启后端容器？
FastAPI 的 `--reload` 模式虽然会自动重启，但 SQLAlchemy 的异步连接池可能不会正确重置，导致数据库连接异常。手动重启容器可以确保所有连接池重新初始化。

### 为什么需要重置数据库？
修改数据库模型后，如果不运行迁移脚本，ORM 模型与数据库表结构不匹配，会导致查询失败。开发环境下直接重置数据库是最快的方式。
