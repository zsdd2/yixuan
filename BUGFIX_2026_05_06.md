## 修复概述

本次修复解决了 4 个影响用户体验的关键 bug，**根本原因是前端代码中使用原生 `fetch` API 时未携带认证 token**，导致后端返回 401 未认证错误。

---

## 问题根源分析

### 技术债务

项目中存在两种 HTTP 请求方式：
1. **封装的 `request` 工具**（`src/api/request.ts`）- 自动注入 token ✅
2. **原生 `fetch` API** - 需要手动注入 token ❌

部分组件为了使用 `Promise.all` 并发请求或避免循环依赖，直接使用了原生 `fetch`，但忘记携带认证 token，导致所有需要认证的接口都返回 401 错误。

## Issue 1: 最终交付图不显示

### 问题描述
- 外面显示有 1 张成片，但点进去显示为空
- 控制台可能显示 401 Unauthorized 错误

### 根本原因
`ProjectDelivery.vue` 第 338 行使用 `fetch` 获取照片列表时未携带 token。

### 修复方案
**文件：** `frontend_admin/src/components/ProjectDelivery.vue:335-347`

```typescript
async function fetchDeliveryPhotos() {
  try {
    const token = localStorage.getItem('token')
    const headers = token ? { 'Authorization': `Bearer ${token}` } : {}

    const [photosRes, targetsRes, projectRes] = await Promise.all([
      fetch(`/api/v1/projects/${props.projectId}/photos?skip=0&limit=500&process_state=final`, { headers }),
      fetch(`/api/v1/projects/${props.projectId}/targets`, { headers }),
      fetch(`/api/v1/projects/${props.projectId}`, { headers }),
    ])
    // ...
  } catch {}
}
```

### 影响接口
- `GET /api/v1/projects/{id}/photos?process_state=final`
- `GET /api/v1/projects/{id}/targets`
- `GET /api/v1/projects/{id}`

## Issue 2: 进度快报显示完成图数量

### 问题描述
进度快报显示"已选 X · 精修 Y · 成片 Z"，但 Y 的数量包含了精修图和完成图，导致统计不准确。

### 根本原因
`ProjectDetail.vue` 第 606 行将 `retouched` 和 `final` 混在一起统计。

### 修复方案
**文件：** `frontend_admin/src/views/ProjectDetail.vue:603-607`

```typescript
// 修复前
retouchedPhotoCount.value = photosData.items.filter((p: any) => 
  p.process_state === 'retouched' || p.process_state === 'final'
).length

// 修复后
retouchedPhotoCount.value = photosData.items.filter((p: any) => 
  p.process_state === 'retouched'
).length
// finalPhotoCount 已经从 targets 聚合数据中获取，无需重复计算
```

**文件：** `frontend_admin/src/views/ProjectDetail.vue:93`

```vue
<!-- 同时修改文案 -->
<div class="metric-sub">已选 {{ selectedPhotoCount }} · 精修 {{ retouchedPhotoCount }} · 完成图 {{ finalPhotoCount }}</div>
```

## Issue 4: 快速分拣图片未同步到子项目

### 问题描述
- 外面显示正常，进入子项目显示"暂无原图"
- 图片分拣进项目但未同步成功

### 根本原因
`TargetDetail.vue` 第 268 行使用 `fetch` 调用批量更新接口时未携带 token。

### 修复方案
**文件：** `frontend_admin/src/components/TargetDetail.vue:264-283`

```typescript
async function movePickedToTarget() {
  const ids = Array.from(pickerSelected)
  if (ids.length === 0) return
  try {
    const token = localStorage.getItem('token')
    const headers: Record<string, string> = { 'Content-Type': 'application/json' }
    if (token) headers['Authorization'] = `Bearer ${token}`

    const res = await fetch('/api/v1/photos/bulk-update', {
      method: 'PATCH',
      headers,
      body: JSON.stringify({ photo_ids: ids, target_id: props.targetId }),
    })
    // ...
  } catch (e: any) {
    ElMessage.error(e.message || '操作失败')
  }
}
```

### 影响接口
- `PATCH /api/v1/photos/bulk-update`

## 技术债务与改进建议

### 1. 统一 HTTP 客户端

**问题：** 项目中混用两种 HTTP 请求方式导致认证逻辑不一致。

**建议：**
- 全面使用封装的 `request` 工具（`src/api/request.ts`）
- 禁止直接使用原生 `fetch` API
- 如需并发请求，在 `request` 工具中添加 `Promise.all` 支持

### 2. 代码审查规则

**建议添加 PR 审查清单：**
- [ ] 检查是否有裸 `fetch(` 调用
- [ ] 确认所有 HTTP 请求都携带认证 token
- [ ] 验证错误处理是否完整

### 3. ESLint 规则

**建议添加规则：**

```javascript
// .eslintrc.js
rules: {
  'no-restricted-globals': ['error', {
    name: 'fetch',
    message: '请使用 src/api/request.ts 中的封装方法，而非直接使用 fetch'
  }]
}
```

### 4. 潜在风险点排查

**搜索项目中所有 `fetch(` 调用：**

